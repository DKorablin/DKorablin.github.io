SAL Host MDI [AvaloniaUI]

{% include files_list.html releases=site.data.Flatbed-MDI-AvaloniaUI %} MDI interface based on modified WPF from AvaloniaUI framework and wieslawsoltes Dock component.
For the time being it's supporting only Windows OS.

Automation

Host will automate following actions:

  1. Size and location of application and all of the opened windows.
  2. Events handling through System.Diagnostics.Trace (All errors will be stored in separate window: View→Event List. All events will be stored at View→Output).
  3. Moving window to working area after (Window.WidthProperty, Window.HeightProperty or Window.WindowStateProperty) event is triggered.
  4. Supported plugins with Windows Forms controls or WPF controls.

Installation

Extract all contents to empty folders, add required plugins to current directory and launch Flatbed.MDI.AvaloniaUI.exe file. If you required to launch separate instance of executable, then you need to add Kernel plugin (For example Kernel.Empty) or change keys SAL_KernelName and SAL_KernelCompany in the Flatbed.MDI.AvaloniaUI.dll.config file. Without changing default kernel plugin name all plugins collection will be combined and identified as default Kernel libraries with single settings storage.

/SAL_path:C:\MyApplication\

Requirements

The application requires .NET version 8 or higher to work.
